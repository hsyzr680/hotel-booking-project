  المتغيرات الثلاثة الموجودة في .env.local
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


1. DATABASE_URL
───────────────────────────────────────────────────────────────────

   الحالة: ❌ يحتاج تعديل

   القيمة الحالية (مثال):
   DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/DATABASE"

   ماذا تفعل؟
   ↓
   احصل على رابط قاعدة بيانات حقيقي من:
   • Supabase (supabase.com) ← مجاني
   • Neon (neon.tech) ← مجاني وسريع

   كيف؟
   ↓
   اقرأ الملف: "كيف_تحصل_على_المتغيرات.md"
   فيه شرح مفصل خطوة بخطوة مع صور


───────────────────────────────────────────────────────────────────


2. NEXTAUTH_SECRET
───────────────────────────────────────────────────────────────────

   الحالة: ✅ جاهز ويعمل!

   القيمة الحالية:
   NEXTAUTH_SECRET="7K9mNpR2tX5wY8zB3cD6fG9hJ2kL5nP8qS1tV4w..."

   ماذا تفعل؟
   ↓
   لا شيء! استخدمه كما هو ✅


───────────────────────────────────────────────────────────────────


3. NEXTAUTH_URL
───────────────────────────────────────────────────────────────────

   الحالة: ✅ جاهز للتطوير المحلي!

   القيمة الحالية:
   NEXTAUTH_URL="http://localhost:3000"

   ماذا تفعل؟
   ↓
   للمحلي: لا شيء! جاهز ✅
   للنشر: غيّره لرابط Vercel

الخطوة 1: احصل على DATABASE_URL
────────────────────────────────────────────────────────────────

  الطريقة الأسهل ← Supabase:

  1. افتح https://supabase.com
  2. Sign up بـ GitHub
  3. New Project
     - Name: hotel-booking
     - Password: اختر كلمة سر (احفظها!)
  4. انتظر 2 دقيقة
  5. Settings → Database → Connection String → URI
  6. Copy
  7. غيّر [YOUR-PASSWORD] بكلمة السر الحقيقية
  8. ضعه في .env.local مكان DATABASE_URL


الخطوة 2: شغّل المشروع
────────────────────────────────────────────────────────────────

  افتح PowerShell في مجلد المشروع:

  npx prisma db push
  npm run db:seed
  npm run dev


الخطوة 3: افتح المتصفح
────────────────────────────────────────────────────────────────

  http://localhost:3000

  تسجيل دخول:
  • Email: admin@test.com
  • Password: admin123
